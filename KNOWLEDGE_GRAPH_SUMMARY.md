# 知识图谱功能实现总结

## 已完成的功能

### 1. 类型定义扩展 (`lib/types.ts`)
- 添加了知识图谱相关的类型定义：
  - `KnowledgeGraphNode`: 图谱节点（论文、概念、作者、方法、数据集、关键词）
  - `KnowledgeGraphEdge`: 图谱边（引用、扩展、使用、相似等关系）
  - `KnowledgeGraph`: 完整的知识图谱结构
  - `AutoGraphAnalysis`: 自动图谱分析结果
  - `GraphBuildRequest/Response`: API请求响应类型

### 2. 知识图谱存储 (`lib/knowledgeGraph.ts`)
- 实现了知识图谱的本地存储（localStorage）
- 提供了完整的CRUD操作：
  - `getKnowledgeGraph()`: 获取图谱
  - `addGraphNode()`: 添加节点
  - `addGraphEdge()`: 添加边
  - `findNodeByLabel()`: 查找节点
  - `findPaperNode()`: 查找论文节点
  - `findRelatedNodes()`: 查找相关节点
  - `updateNodeWeight()`: 更新节点权重
  - `cleanupOrphanNodes()`: 清理孤立节点

### 3. 自动图谱构建 (`lib/autoGraph.ts`)
- 实现了自动图谱构建功能：
  - `autoGraphBuild()`: 使用小参数模型自动分析论文并构建图谱
  - `shouldAutoGraphBuild()`: 判断是否需要构建图谱

### 4. API接口

#### 分析接口 (`app/api/knowledge-graph/analyze/route.ts`)
- 使用小参数模型分析论文
- 提取核心概念（2-4个）
- 提取关键方法（1-3个）
- 自动生成标签（4-6个）
- 分析与现有论文的潜在关联
- 改进的JSON解析，支持容错和自动修复

#### 构建接口 (`app/api/knowledge-graph/build/route.ts`)
- 根据分析结果创建图谱节点和边
- 自动创建论文节点
- 创建作者节点和关系
- 创建概念节点和关系
- 创建方法节点和关系
- 创建关键词节点和关系
- 建立论文间的关联关系
- 自动更新知识库条目的标签

### 5. 知识图谱可视化页面 (`app/knowledge-graph/page.tsx`)
- 使用 ReactFlow 实现图谱可视化
- 支持节点拖拽和缩放
- 不同类型节点使用不同颜色：
  - 论文：蓝色
  - 概念：绿色
  - 作者：橙色
  - 方法：紫色
  - 数据集：粉色
  - 关键词：灰色
- 节点大小根据权重动态调整
- 边的粗细根据关系强度调整
- 点击论文节点可跳转到沉浸式阅读
- 显示图谱统计信息（节点数、边数）
- 提供图例说明

### 6. 知识库面板集成 (`components/Knowledge/KnowledgePanel.tsx`)
- 添加了"图谱"按钮，可快速跳转到知识图谱页面
- 添加了 GraphIcon 图标组件

### 7. 沉浸式阅读集成 (`app/immersive/[id]/page.tsx`)
- 在PDF解析完成后自动触发图谱构建
- 在已有缓存的情况下也会触发图谱构建
- 使用小参数模型处理，不影响阅读体验
- 后台异步执行，不阻塞用户操作

## 功能特点

### 智能分析
- 使用小参数模型进行分析，成本低、速度快
- 自动提取论文的核心概念和方法
- 智能生成相关标签
- 基于作者、关键词、标题相似性分析论文关联

### 自动化
- 每次阅读新论文时自动构建图谱
- 自动创建节点和关系
- 自动更新论文标签
- 不需要用户手动操作

### 智能关联
- 不强行建立关联，只在置信度足够高时才创建边
- 支持多种关系类型（引用、扩展、相似等）
- 关系强度可量化（0-1）
- 自动清理孤立节点

### 可视化
- 直观的图谱展示
- 支持交互操作
- 节点大小反映重要性
- 边的粗细反映关系强度
## 使用流程

1. 用户上传或导入论文到知识库
2. 进入沉浸式阅读模式
3. 系统自动解析PDF并提取元数据
4. 后台自动调用小参数模型分析论文
5. 自动创建图谱节点（论文、作者、概念、方法、关键词）
6. 自动建立节点间的关系
7. 自动为论文添加标签
8. 用户可以点击"图谱"按钮查看完整的知识图谱
9. 在图谱中点击论文节点可跳转到沉浸式阅读

## 技术实现

- **前端**: React + @xyflow/react (ReactFlow v12+)
- **存储**: localStorage（本地存储）
- **AI模型**: 使用用户配置的小参数模型
- **API**: Next.js API Routes
- **类型安全**: 完整的TypeScript类型定义

## 注意事项

1. 需要配置小参数模型才能使用自动图谱功能
2. 图谱数据存储在本地浏览器中
3. 首次构建图谱需要一定时间（取决于论文数量）
4. JSON解析支持容错，即使AI返回不完整也能处理
5. 需要安装 `@xyflow/react` 依赖包才能使用可视化功能（新版ReactFlow）
